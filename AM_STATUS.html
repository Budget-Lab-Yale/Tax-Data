Tilt Wealth Imputation — v3 Report

Method

For each PUF tax unit, we ask the per-(income-cell) DRF for forest- averaged donor probabilities p_ij over the cell’s SCF bootstrap, then tilt them by q_ij ∝ p_ij · exp(Z_jᵀ λ) where Z_j are donor-side wealth target features (8 categories × {amount, count}) and λ is optimized per (income × age) bucket against SCF cell aggregates via BFGS. Solver runs on a normalized basis (Z_n = Z / max|Z| per column) so condition number stays in check across count vs amount targets. One donor per record is then sampled from q_ij with top-K=300 sparsification.

A residual rescale (Step B) closes any per-(cell × cat) dollar gap that the tilt couldn’t reach. Three modes:

  • multiplicative — factor = SCF_total / PUF_total when both alive
  • fallback_uniform — when tilt collapsed donors to zero in a category (PUF total ≈ 0 but SCF target > 0), distribute SCF aggregate uniformly across PUF records in the cell
  • skip — when SCF target ≈ 0 (don’t fabricate amounts)

DRF min.node.size = 50. Per-cell forests cached at resources/cache/qrf/wealth_percell_<ci>_mns50.rds.
